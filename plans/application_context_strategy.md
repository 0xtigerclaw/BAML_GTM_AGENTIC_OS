# Plan: Application Context Intelligence

## The Objective
Ensure every essay generated by Porter is deeply aligned with the specific nature, values, and target demographics of the application (e.g., YC vs. NSF Grant vs. Corporate Pilot).

## The Strategy: "Recon & Execute" (Split Responsibility)

Instead of spinning up a new agent for *every* single essay question (which risks memory overload and high latency), we implement a **One-Time Reconnaissance Step**.

### Phase 1: Reconnaissance (Curie / Scout)
**Role:** Strategic Analyst
**Timing:** Once per form template (at scan time or on demand).
**Task:**
1.  **Scan**: Analyze the application URL, the "About" page, and "Evaluation Criteria".
2.  **Profile**: Generate an `ApplicationProfile` containing:
    -   **Archetype**: VC (Growth), Grant (Impact), Corp (Synergy), Hackathon (Speed).
    -   **Audience Persona**: "Impatient VC Associate" vs "Academic Review Board".
    -   **Key Values**: e.g., "Traction", "Novelty", "DEI", "Sustainability".
    -   **Demographic Focus**: Specific requirements (e.g., "Underrepresented Founders").
3.  **Store**: Save this DNA into the `form_templates` database table.

### Phase 2: Execution (Porter / Writer)
**Role:** Tactical Writer
**Timing:** Per essay question.
**Task:**
1.  **Context Injection**: Retrieve the standard RAG context (Company Info).
2.  **Profile Injection**: Retrieve the `ApplicationProfile` (Target Info).
3.  **Synthesis**: Write the essay combining *Who We Are* (RAG) with *What They Want* (Profile).

## Technical Implementation

### 1. Schema Update (`convex/schema.ts`)
Add `contextProfile` field to `form_templates`:
```typescript
contextProfile: v.optional(v.object({
  archetype: v.string(), // "Accelerator", "Grant", ...
  audience: v.string(),
  values: v.array(v.string()),
  tone: v.string(),
  demographics: v.optional(v.string())
}))
```

### 2. Recon Action (`convex/forms.ts`)
New action `analyzeApplicationContext(url)`:
- Fetches page content.
- Calls LLM to extract the profile elements.
- Updates the template.

### 3. Updated Generation (`convex/forms.ts`)
Modify `generateEssayResponse`:
- Read `contextProfile` from the template.
- Append a "Target Audience Directive" to the Porter task prompt.

## Consequences & Trade-offs

| Approach | Latency | Memory Impact | Consistency | Verdict |
|----------|---------|---------------|-------------|---------|
| **Dedicated Agent (Per Question)** | High (2x) | **High (Risk)** | Varible | ❌ Too heavy |
| **Manual User Input** | Low | Low | High | ❌ High friction |
| **One-Time Recon (Recommended)** | **Medium (Once)** | **Low** | **Highest** | ✅ Best Balance |

This approach keeps the "Memory Safe" promise (1 agent at a time) while adding the "High Intelligence" layer (deep context awareness).
